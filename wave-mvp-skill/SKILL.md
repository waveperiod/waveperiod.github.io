# Wave GAS Fitness Planner - Main Skill

## 项目概述
5天开发MVP验证网站：12周GAS训练 + 4周月经周期可视化

---

## 🎯 工作模式：边做边探索

### 核心原则
- **不预先定义所有细节**
- **在实现过程中逐步完善**
- **根据实际情况调整方案**

### 我的角色
- 提供专业建议，但不强制方案
- 问关键问题帮你思考
- 记录决策和原因

## 🚀 如何开始

### Step 1: 明确今天要做什么
```
"今天我想做XX功能，给我建议"
"作为PM，这个功能优先级是P几？"
```

### Step 2: 选择角色开始工作
```
"作为开发，实现这个功能"
"作为设计师，这个交互怎么做？"
```

### Step 3: 遇到问题随时提问
```
"这个技术方案有什么替代方案？"
"用户可能在这里卡住，怎么优化？"
```

### Step 4: 记录决策（我会帮你）
```
"把今天的决策记录到features.md"
"更新dev-plan.md的进度"
```

## 📋 模块说明

### brand.md
**什么时候看**：
- 设计页面时

- 需要确认视觉风格时

**包含内容**：
- Logo规范
- 配色系统
- 文案原则
- 品牌调性

---

### tech-stack.md
**什么时候看**：
- 开始写代码前
- 遇到技术选型问题时
- 需要找库/工具时

**包含内容**：
- 推荐技术栈
- 优缺点分析
- 常用库链接
- 部署指南

---

### features.md
**什么时候看**：
- 不确定做什么时
- 需要评估优先级时
- 想看整体进度时

**包含内容**：
- P0/P1/P2功能列表
- 每个功能的状态（待做/进行中/完成）
- 简要说明
- **会随着开发动态更新**

---

### roles.md

- 每个角色会问的问题
- 输出格式

---

### dev-plan.md
**什么时候看**：
- 每天开始工作时

- **会根据实际进度更新**

## 🔄 迭代方式

### 功能迭代
```
Day 1: 做最简单的版本（能跑起来）
Day 2: 加核心功能
Day 3: 优化体验
Day 4: 打磨细节
Day 5: 上线
```

### 模块迭代
```
发现新问题 → 讨论方案 → 更新对应模块 → 继续开发
```
